# JarvisFather — План реализации

## Этап 0: Инфраструктура (1-2 дня)

- [ ] Создать структуру репозитория
- [ ] Настроить виртуальное окружение (Python 3.12, uv или poetry)
- [ ] Docker Compose: PostgreSQL + pgvector, Redis
- [ ] Alembic для миграций БД
- [ ] `.env` конфигурация (токен бота, ключ шифрования, DSN БД, Redis URL)
- [ ] Базовый `config.py` с pydantic-settings

---

## Этап 1: Центральный бот — скелет (2-3 дня)

- [ ] Инициализация aiogram 3.x (dispatcher, router, FSM storage → Redis)
- [ ] Обработчик `/start` — приветствие
- [ ] FSM для интервью (состояния: бизнес → задачи → подтверждение → credentials)
- [ ] Модели БД: `users`, `user_servers`, `deployments`
- [ ] Зашифрованное сохранение credentials (Fernet)

---

## Этап 2: Интервью и подбор агентов (3-4 дня)

- [ ] Логика интервью: 5-7 вопросов с inline-кнопками
- [ ] Базовый каталог агентов и скилов (10-15 штук в markdown)
- [ ] Простой подбор по ключевым словам (без RAG — для MVP)
- [ ] Показ пользователю что будет установлено + подтверждение
- [ ] Запрос и валидация данных: IP, SSH, токен бота, OAuth URL
- [ ] Извлечение кода авторизации из Claude OAuth URL

---

## Этап 3: Деплой через SSH (4-5 дней)

- [ ] Celery + Redis: базовая очередь задач
- [ ] SSH клиент на Paramiko: подключение, выполнение команд, SFTP
- [ ] Bash-скрипт установки на VPS:
  - [ ] apt: nodejs, npm, git, python3, pip
  - [ ] Claude Code CLI + OAuth авторизация
  - [ ] Копирование ~/.claude/ (агенты, скилы, хуки)
  - [ ] Копирование aiogram бота на VPS
  - [ ] systemd user service для бота
- [ ] Уведомления пользователю: старт → прогресс → успех/ошибка
- [ ] Retry-логика (3 попытки при ошибке)
- [ ] Логирование деплоя в таблицу `deployments`

---

## Этап 4: Бот на VPS пользователя (3-4 дня)

- [ ] Шаблон aiogram бота (минималистичный)
- [ ] SQLite + aiosqlite: хранение истории разговоров
- [ ] Вызов Claude Code в headless режиме (`claude -p`)
- [ ] Передача контекста (последние N сообщений) в промт
- [ ] Обработка ошибок Claude (таймаут, недоступность)
- [ ] systemd unit файл для автозапуска

---

## Этап 5: RAG система (3-4 дня)

- [ ] pgvector: включение расширения, миграция таблицы `configurations`
- [ ] Генерация эмбеддингов (sentence-transformers локально или Claude API)
- [ ] Индексация успешных конфигураций после деплоя
- [ ] Поиск похожих конфигураций при новом интервью
- [ ] Замена ключевого поиска (Этап 2) на RAG-подбор
- [ ] Метрика успешности: активность бота через 7 дней

---

## Этап 6: Freemium (2-3 дня)

- [ ] Поле `tier` в модели `users` (free / pro)
- [ ] Каталог: пометить агенты как free/pro
- [ ] Блокировка pro-агентов для free пользователей
- [ ] Заглушка оплаты (Telegram Stars или ЮKassa — выбрать)
- [ ] Апгрейд тарифа → повторный деплой с новыми агентами

---

## Этап 7: Тестирование и запуск MVP (3-4 дня)

- [ ] Тест деплоя на чистый Beget VPS (Ubuntu 22.04)
- [ ] Тест деплоя на чистый TimeWeb VPS
- [ ] Нагрузочный тест: 10 параллельных деплоев через Celery
- [ ] Обработка edge-cases: неверный IP, закрытый SSH порт, нет Claude подписки
- [ ] Документация для первых пользователей (что нужно подготовить перед стартом)

---

## Порядок приоритетов

```
Этап 0 → 1 → 2 → 3 → 4 → тест MVP → 5 → 6 → 7
```

Этапы 3 и 4 можно вести параллельно (разные части системы).
